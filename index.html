빈 코드는 저장되지 않습니다. 정확 → 공백무시 → 부분 포함 순으로 매칭됩니다.
배송처 ↔ 이카운트 거래처 매핑 (구매입력용)
플레이오토 배송처(depot)를 이카운트 거래처코드에 연결합니다. 구매입력 시 거래처 자리에 들어갑니다.
💡 키는 배송처명(예: 지플랜) 또는 PA depot_no(예: 179937) 어느 쪽이든 사용 가능합니다.
매칭 순서: 정확일치 → 공백제거 → 부분포함.
SKU별 택배발송비 매핑 (구매입력 택배비용 단가)
품목(SKU)마다 택배발송비가 다르면 여기서 SKU 코드별 단가를 지정하세요. 매칭된 값이 있으면 설정의 '기본 택배발송비'보다 우선합니다.
💡 우선순위: SKU별 매핑 → 설정의 '기본 택배발송비' → PA 주문 `ship_cost`.
SKU 코드는 이카운트 전송 시 사용되는 품목코드(예: K10030, C0001)와 동일해야 합니다.
📥 PA 엑셀 업로드: PA 관리자 → SKU상품관리 → 엑셀 메뉴에서 내려받은 파일을 선택하면 'SKU코드' + '택배발송비' 컬럼을 자동 매핑합니다.

📖 복사 방법 (클릭해서 보기)
  1. 브라우저에서 네이버 쇼핑 검색 (예: search.shopping.naver.com/search/all?query=캠핑의자)
  2. F12 눌러 개발자도구 열기 → Network 탭
  3. 왼쪽 필터에 all 또는 search/all 입력
  4. 페이지 새로고침 (F5)
  5. 목록에서 all 항목 클릭 → 우측 Response 탭 클릭
  6. Response 영역 우클릭 → Copy value (또는 내용 드래그 후 Ctrl+C)
  7. 위 텍스트 박스에 붙여넣기 (Ctrl+V) → "manuTag 추출" 클릭
